# Neurodivergent UX Documentation

## Overview

Lumo is designed for neurodivergent users with emotional safety, calm UX, and low cognitive load as core principles. The app reduces overwhelm, supports inconsistent energy, and adapts to cognitive load.

## Philosophy

### Emotional Safety

The app should feel like:
- A calm workspace
- A supportive companion
- An emotionally safe environment

NOT:
- Productivity pressure software
- Gamified achievement systems
- Dopamine extraction software

### Core Principles

- **Reduce overwhelm**: Fewer visible decisions
- **Support inconsistent energy**: Adapt to capacity
- **Adapt to cognitive load**: Dynamic complexity
- **Preserve emotional safety**: No guilt, no urgency

## Cognitive Load Management

### Visible Decisions

Reduce simultaneous decisions:
- Progressive disclosure
- Contextual surfacing
- Adaptive layouts
- Reduced interruption patterns

### Dashboard Complexity

Dynamically reduce complexity:
- Fewer visible cards
- Fewer simultaneous actions
- Reduced visual hierarchy depth
- Minimized optional actions

### Interaction Density

Lower interaction density when needed:
- Simplified mode
- Focus modes
- Cognitive load adaptation
- Sensory-safe patterns

## Sensory Safety

### Visual Safety

- Soft gradients
- Spacious layouts
- Calm hierarchy
- Predictable interactions

### Motion Safety

- Optional animations
- Reduced motion mode
- Intensity scaling
- No forced transitions

### Haptic Safety

- User-controlled intensity
- Optional feedback
- Centralized management
- Accessibility-aware defaults

## Adaptive Complexity

### Simplified Mode

When enabled:
- Reduced UI density
- Fewer visible actions
- Simplified dashboard
- Lower visual complexity
- Calmer layouts

### Cognitive Load Adaptation

Automatic adjustment based on:
- Visible action count
- Interaction frequency
- Dashboard density
- Time spent in app

### Focus Modes

Adaptive modes for different needs:
- Single task mode for deep focus
- Distraction-free for reduced stimulation
- Today-only for daily focus
- Calm mode for low-energy states

### Overstimulation

## UX Patterns

### Progressive Disclosure

- Show information when needed
- Hide complexity until relevant
- Provide clear paths forward
- Reduce initial cognitive load

### Contextual Surfacing

- Show relevant actions only
- Hide irrelevant options
- Adapt to current context
- Respect user intent

### Adaptive Layouts

- Adjust to screen size
- Respect font scaling
- Adapt to preferences
- Preserve layout stability

## Emotional Design

### Supportive Messaging

- "A gentle place to begin"

- "You're doing great."

### Forgiving Systems

- No punishment for failures
- Easy recovery from errors

### Calm Recovery
